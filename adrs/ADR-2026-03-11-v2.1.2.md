**Architectural Decision Record (ADR-2026-03-11-v2.1.2)**  
**Title:** User-Agent Skill – Mandatory Paranoid Safety Layer (Threat-Model-First ReAct)  
**Status:** Approved – non-negotiable for v2.1.1+  
**Rationale (maniacal security):** Prompt injection, jailbreaks, social-engineering tricks, and ambiguous user intent are the #1 attack vector once natural-language input enters the system. The `user-agent` skill is the only place where untrusted user text meets LLM tool-calling. Therefore it MUST implement a hardened, auditable safety gate **before** any skill is invoked.  

**Selected approach (most secure + still user-friendly):**  
- Every user request triggers a **mandatory 2-phase ReAct loop** inside `user-agent`:  
  1. **Threat-model phase** (always first, zero-shot with fixed safety system prompt).  
  2. **Execution phase** (only if user explicitly approves after seeing concerns).  
- Safety system prompt is **immutable** (baked into the skill’s Go code, never generated or user-modifiable).  
- Every reasoning step, every detected risk, and every user confirmation is forwarded to `message-hub` → seedclaw binary → immutable audit log (with full `network_policy`).  
- High-risk actions (network, code exec, file writes, external calls) **require explicit user “YES”** before the tool call is issued. Low-risk actions may proceed with a summary warning.  
- This preserves Principle #7 (explicit minimal network surface) and trivial auditing: `grep -E 'threat_model|risk|confirmation|injection' shared/audit/seedclaw.log` shows every safety decision ever made.  

**Why this beats alternatives**  
- Pure LLM tool-calling without safety gate → instant jailbreak surface. Rejected.  
- Auto-execute everything → violates zero-trust. Rejected.  
- Binary-level safety checks → TCB bloat. Rejected.  
- Result: user sees concerns in real time (“This prompt looks like a classic DAN-style jailbreak attempt. Proceed anyway? (YES/NO)”), applies human judgement, everything audited.  

**Security invariants enforced in code (will be in generated user-agent.go):**  
- Safety prompt is a compile-time constant string.  
- No tool call is emitted until `user-agent` receives a JSON confirmation from seedclaw (originating from STDIN).  
- Any attempt to bypass the safety phase (via prompt injection) is logged as `action: "safety_violation_attempt"`.
